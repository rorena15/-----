# 프로젝트 Z-SNAP (인생네컷 메이커) 인수인계서

## 1. 프로젝트 개요

- **프로젝트명**: Z-SNAP (구 인생네컷 메이커)
- **목적**: 학교 행사 및 축제용 웹 기반 인생네컷 촬영 및 공유 서비스
- **핵심 가치**: 광고 없는 청정 QR 다운로드, 고퀄리티 디자인 프레임 오버레이, 보안 강화형 이미지 호스팅

## 2. 현재 개발 상태 (Phase 1 & 2 완료)

- [x] **하이브리드 서버 인프라**: XAMPP(PHP) + ngrok(터널링) 환경 구축 완료
- [x] **지능형 QR 시스템**: 접속 환경(Local IP 또는 ngrok URL)을 자동 감지하여 QR 주소 생성
- [x] **고정 레이아웃 시스템**: 사진 위치를 고정하고 PNG 오버레이만 교체하는 프리셋 구조
- [x] **보안 엔지니어링**: 파일 업로드 취약점 방어 및 자동 파기 로직 적용
- [x] **운영 자동화**: `서버실행기.bat`를 통한 원클릭 서버 가동 환경 구축

## 3. 기술 스택 및 구조

- **Frontend**: HTML5, CSS3, Vanilla JavaScript
  - `life4cuts_v4.html`: UI 구조 및 단계별 페이지 구성
  - `main_css.css`: 프레임 갤러리 및 모바일 반응형 디자인
  - `life4.js`: WebRTC 카메라 제어, Canvas API 이미지 합성, 스티커 드래그
- **Backend**: PHP 7.4+ (XAMPP)
  - `upload.php`: 이미지 수신, 보안 검증, 재인코딩, URL 반환
- **Infrastructure**: ngrok (외부 터널링), Windows Batch (자동화)

## 4. 핵심 보안 구현 사항

- **이미지 재인코딩 (GD Library)**: 업로드된 바이너리를 `imagecreatefromstring`으로 재생성하여 메타데이터 내 악성 스크립트 제거
- **실행 권한 차단**: `uploads/` 디렉토리 내 `.htaccess` 설정을 통해 PHP 등 스크립트 실행 원천 봉쇄
- **자동 파기**: `cleanup.php` 로직을 통해 생성 후 1시간이 지난 이미지는 서버에서 자동 삭제
- **파일명 난독화**: `bin2hex(random_bytes(16))`를 사용해 파일 경로 추측 공격(IDOR) 방지

## 5. 프레임 디자인 가이드 (디자이너 전달용)

- **규격**: 300px (가로) x 680px (세로)
- **구조**: 상단/좌/우 20px 여백, 사진 간격 10px, 하단 로고 영역 80px
- **포맷**: 사진 구멍이 투명하게 뚫린 **24-bit PNG**
- **적용**: `frames/` 폴더에 저장 후 `life4.js`의 `FRAMES` 객체에 경로 매핑

## 6. 향후 로드맵 (Phase 3 & 4)

1. **모바일 최적화**: iOS/Android 기기별 캔버스 스케일링 및 터치 드래그 감도 미세 조정
2. **사운드 이펙트**: 촬영 시 셔터 사운드 및 단계별 효과음 추가
3. **부하 테스트**: ngrok 무료 티어의 트래픽 제한 확인 및 대규모 접속 대비 안정화
4. **UI 고도화**: 디자인 썸네일 실제 적용 및 폰트 라이브러리 추가

## 7. 로컬 개발 환경 세팅 방법

1. **폴더 연결**: `mklink /D "C:\xampp\htdocs\min" "E:\Shingu_File\민영이누나"` 명령어로 작업 폴더 링크
2. **서버 실행**: `서버실행기.bat` 실행 (PHP 서버 + ngrok 자동 가동)
3. **접속**: 화면에 표시된 `https://[ngrok-id].ngrok-free.app/min/index.html`로 접속

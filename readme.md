# 학교 행사용 인생네컷 메이커 (Life4Cuts Maker)

학생들이 별도의 앱 설치 없이 웹 브라우저를 통해 네컷 사진을 촬영하고, 광고 없이 깔끔하게 결과물을 소장할 수 있도록 돕는 웹 애플리케이션입니다.

---

## 🛠 1. 핵심 개발 및 운영 정책

### 1.1 광고 없는 청정 QR 정책 (Clean QR Policy)
사용자 경험을 저해하는 외부 광고를 완전히 배제하고 사진 파일에만 집중합니다.
- **다이렉트 링크:** QR 스캔 시 광고가 포함된 랜딩 페이지를 거치지 않고, 이미지 파일(`.png`, `.jpg`)로 직접 연결되는 직링크(Direct Link) 방식을 채택합니다.
- **호스팅 선정:** 광고 삽입이 잦은 무료 임시 호스팅(tmpfiles 등)을 배제하고, Imgur API나 전용 이미지 프록시 서버를 활용하여 깨끗한 다운로드 환경을 제공합니다.

### 1.2 비주얼 프레임 및 에셋 정책 (Frame & Asset Policy)
사용자가 시각적으로 디자인을 선택하고 높은 퀄리티의 결과물을 얻을 수 있도록 합니다.
- **PNG 오버레이 방식:** 코드상에서 도형을 그리는 방식이 아닌, 전문 디자인된 투명 배경(PNG) 프레임을 사진 위에 덮어씌워 퀄리티를 높입니다.
- **갤러리 UI:** 텍스트 버튼 대신 실제 프레임 디자인의 썸네일을 보고 선택할 수 있는 직관적인 디자인 갤러리 인터페이스를 제공합니다.
- **커스텀 최적화:** 학교 로고, 행사 슬로건 등이 포함된 '행사 한정 프레임'을 손쉽게 추가할 수 있는 구조를 유지합니다.

### 1.3 현장 UX 최적화 정책 (Event UX Policy)
축제 현장의 혼잡함을 고려하여 빠르고 간편한 사용성을 보장합니다.
- **모바일 우선(Mobile First):** 스마트폰 환경에서의 촬영 및 QR 저장 경험을 최우선으로 최적화합니다.
- **최소 단계:** [촬영 -> 셀렉 -> 꾸미기 -> 저장]으로 이어지는 흐름을 방해 요소 없이 선형적으로 구성합니다.
- **개인정보 보호:** 촬영된 사진은 저장 및 다운로드를 위한 용도로만 일시 사용되며, 일정 시간 후 서버에서 자동 삭제되도록 관리합니다.

---

##  2. 기술 스택 (Technical Stack)

- **Frontend:** HTML5, CSS3, Vanilla JavaScript
- **Graphics:** HTML5 Canvas API (이미지 병합 및 오버레이)
- **Camera:** WebRTC API
- **QR Code:** QRCode.js
- **Storage/API:** Imgur API 또는 전용 이미지 호스팅 (광고 없는 직링크 지원용)

---

##  3. 향후 개발 로드맵

1.  **Phase 1 (QR 시스템 개선):** 기존 광고 포함 호스팅 제거 및 Imgur/Cloudinary 등 Clean API 연동
2.  **Phase 2 (프레임 시스템 개편):** PNG 이미지 기반 오버레이 로직 구현 및 썸네일 선택 UI 적용
3.  **Phase 3 (디테일 최적화):** 모바일 터치 드래그 감도 개선 및 현장 배포용 안정화 작업
